1. 기술 산업의 승부는 스펙 싸움이 아니다
(1) 엔비디아가 GPU 벤치마크에 이겨서 AI 시대에 승리한 것이 아님. CUDA로 GPU를 편하게 쓸 수 있는 컴퓨팅 플랫폼으로 바꿈
(2) Stripe는 은행보다 API가 빨라서 성공한 게 아니라, 7줄의 코드로 결제를 붙이게 해줌
(3) 애플의 아이폰은 CPU 성능을 포기하고 발열, 디자인, 전체 시스템 효율을 함께 최적화함

이들이 이긴 이유는 스펙이 더 좋아서가 아니라, 자기 기술이 도입되는데 제약을 정확히 이해하고, 그에 맞는 솔루션을 같이 제공했기 때문
L1/L2의 TPS 경쟁, 가스비 경쟁, 합의 방식, 블록 크기 등의 블록체인 씬에서 논의되는 스펙 경쟁에서 벗어나야함

2. 블록체인이 못하는 것 (source)
(1) Non-determinism (파일 읽기/쓰기, Ai inference, OS 기능 전반)
(2) External I/O (외부 통신 불능)
(3) Event-driven execution 부재 (트잭 안넣으면 상태변경x)
(4) Integrability (외부 통합 불능)

TPS와 블록크기, 낮은가스비 등의 성능 경쟁은 본질적인 해결책이 아님. 운영체제 자체가 극도로 제한적인 상태에서 CPU나 메모리만 늘리는 것. 
차도 안다니는 도로에 자꾸 삐까 뻔쩍한 아스팔트 깔 필요 없음

3. 블록체인이 정말 의미 있는 상황 (기존 컴퓨터는 못하는 것)
① 여러 독립 주체가 하나의 장부를 공유해야 할 때 —->A single source of truth, 탄소배출권, 글로벌 공급망, 국경간 금융 인프라
② 운영자를 신뢰하지 않아도 되는 시스템이 필요할 때 —->프로듀스 101 투표 조작 사례, Don't be Evil 에서 Can't be Evil
③ 자산의 상호운용성 또는 조합성(composability)이 핵심일 때 —> 게임간 아이템 이동, 여러 앱에서 포인트 이동, 하나의 "state"가 여러 앱에서 사용

블록체인의 본질 기능에 해당 되는 부분만 블록체인에 저장, 나머지 대부분의 기능은 기존 컴퓨터와 서버에서 처리해야 기존 서비스와 경쟁 가능 (웹3안에서 경쟁은 노의미)
결국 이 본질 기능이 유저를 모으거나, 매출을 높이거나, 비용을 줄일 것으로 기대되야 "블록체인 서비스"로써 의미가 있을 듯
